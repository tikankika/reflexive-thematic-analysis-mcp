# Coding Protocols

This directory contains project-specific coding protocols — operational documents that define how to apply the methodology (described in `../methodology/`) to particular research projects.

## Why these documents are in Swedish

The protocols guide AI-augmented coding of Swedish-language focus group transcripts. They instruct the AI to preserve Swedish in-vivo expressions, use Swedish code names, and write reflexive notes grounded in participants' own language. Translating the protocols into English would create an incoherent workflow: English instructions directing Swedish-language coding of Swedish data.

The methodology documents (in `../methodology/`) describe the analytical framework in English. The protocols operationalise that framework for specific Swedish-language research projects.

## Files

| Protocol | Research focus | Structure |
|---|---|---|
| `coding_protocol_disruptive_3rq.md` | Three integrated research questions on generative AI as disruptive technology in Swedish upper secondary education | Two-level coding (semantic + latent) across RQ1 (student AI use), RQ2 (teacher practice), RQ3 (learning impact) |
| `coding_protocol_sensemaking_v1.md` | Teacher sensemaking around knowledge, learning, and assessment | Two-level coding with epistemic lens — how teachers construct meaning around what AI does to knowing, learning, and assessing |

## Creating new protocols

Protocols are project-specific. To create one for a new research project, define: the research question(s), the epistemic lens, the coding structure (levels, tags, format), and the AI collaboration workflow. Use the existing protocols as examples, adapting language to match the data being analysed.

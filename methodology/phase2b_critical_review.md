# Phase 2b: Critical Coding Review

## Purpose

Phase 2b is a **researcher-driven critical review** of semi-automated coding. It occurs between Phase 2a (initial coding) and Phase 3 (theme generation).

## Why Phase 2b?

When AI assists with initial coding (Phase 2a), the researcher must actively review and validate all codes before proceeding to theme generation. This is not a rubber-stamp process - it requires genuine critical engagement.

### Goals

1. **Verify segment boundaries** - Are segments coherently bounded?
2. **Validate codes** - Are codes accurate and sufficient?
3. **Refine coding** - What codes should be added/removed/refined?
4. **Document reflexivity** - What analytical observations emerge?

## Key Questions

For each coded segment, ask:

### 1. Segment Boundaries
- Is this segment a coherent semantic unit?
- Should it be split into smaller segments?
- Should it be merged with adjacent content?

### 2. Code Accuracy
- Do the codes capture the meaning of this segment?
- Are any codes misapplied or too broad?
- Are there obvious meanings that are not coded?

### 3. Code Sufficiency
- Are there additional codes needed?
- Are there latent meanings not captured?
- Does this segment connect to other coded content?

### 4. Analytical Observations
- What patterns am I noticing? (without premature thematizing!)
- What surprises me in this segment?
- How does this connect to my research questions?

## Workflow

```
Phase 2a: Semi-automated coding (AI proposes codes)
    |
    v
Phase 2b: CRITICAL REVIEW (researcher reviews actively)
    - Segment boundaries correct?
    - Codes accurate and sufficient?
    - Add/remove/refine codes
    - Reflexive note about process
    |
    v
Phase 3: Generating themes
```

## Reflexive Note Format

For each reviewed segment, document:

```markdown
## WHAT HAPPENS IN THE SEGMENT?
[Brief description of content]

## WHAT DO I NOTICE?
[Analytical observations - patterns, surprises, connections]
[Coding decisions and motivations]

## CHANGES MADE
[List any codes added/removed/modified]
```

## Important: Avoid Premature Thematization

Phase 2b is about **reviewing coding**, NOT about grouping codes into themes. That is Phase 3.

### Signs of Premature Thematization

- "This connects with X which forms a theme about..."
- "I see a pattern I'll call Y..."
- "These codes cluster around Z..."

### Correct in Phase 2b

- "I notice several segments mention Z" (observation)
- "Code X recurs - worth noting" (pattern awareness)
- "This segment has similar content to segment 45" (connection)

## Best Practices

1. **Don't rush** - Critical review takes time
2. **Be willing to disagree** - AI's initial codes are proposals, not decisions
3. **Document everything** - Your reflexive notes are part of the audit trail
4. **Stay at the code level** - Save thematic clustering for Phase 3
5. **Trust your interpretation** - You know your data and research context

## Connection to Methodological Integrity

Phase 2b ensures:

- **Credibility** - Codes accurately represent data
- **Dependability** - Coding process is documented and reviewable
- **Confirmability** - Researcher decisions are transparent
- **Reflexivity** - Interpretive process is visible

## Grounding in Theoretical Framework

### How Constructionist Epistemology Shapes Coding

- **Meaningfulness matters more than frequency**
- **Degrees of conviction matter** — strong emotion may be more significant than casual repetition
- **Context shapes meaning** — attend to how language constructs meaning

### How Experiential Orientation Shapes Coding

- Prioritize participants' own meaning-making
- Focus on "meanings made" rather than imposing external frameworks
- Honor participant voice

### How Predominantly Inductive Approach Shapes Coding

- Open-code: allow codes to emerge from participants' language
- Research questions provide relevance frame, not predetermined codes
- Some deductive element inevitable and appropriate

### Semantic and Latent Coding

**Semantic codes**: Explicit, surface meanings (descriptive)
**Latent codes**: Underlying meanings, assumptions, ideologies (interpretive)

The same data can be coded both ways.

**Example** (Byrne):
> "...if someone's struggling...it's common sense that determines what we say to them...And it might help to talk, but I don't know that it has a lasting effect"

- **Semantic**: "educators rely on common sense when attending to wellbeing issues"
- **Latent**: "common sense inadequate for wellbeing promotion"

---

## What Makes a "Good" Code?

✓ Brief but sufficiently detailed to stand alone
✓ Captures underlying commonality across data items
✓ Relevant to research questions
✓ Applies to multiple data items (but meaningfulness > frequency)

**Poor code**: "positivity regarding the guidelines" (whose positivity?)
**Better code**: "educator positivity regarding the wellbeing curriculum"

---

## The Recursive Nature of Coding

Codes WILL evolve. You WILL return to earlier transcripts. This IS the method.

Track and document changes for transparency.

---

## Two Viable Approaches

### Approach A: Human-First Coding (Traditional)

1. **Systematic Human Coding**
   - Work through transcript systematically
   - Use hashtag format: `#code_name__lins1`
   - Preserve Swedish in vivo codes: `#"fulanvändning"__lins1`

2. **Multiple Iterations** (minimum three passes)
   - First: Initial open coding
   - Second: Consolidation and refinement
   - Third: Quality and theory check

3. **AI for Refinement** (optional)
   - Gap identification
   - Consistency verification
   - Latent dimension exploration

### Approach B: AI-First Coding

1. **Orient AI to theoretical framework**
2. **AI generates systematic codes**
3. **Critical Human Evaluation** (THE ESSENTIAL WORK)
4. **Add human interpretive layer**
5. **Iterative refinement**

## Your Coding Conventions

```markdown
[Transcript section]
#misstanke_om_äkthet__lins1 (semantic)
#"skrivbordsprodukt"__lins1 (in vivo)
#→underliggande_oro__lins2 (latent, marked with →)
```

**Format**:
- Swedish language codes
- Hashtags: `#code_name`
- Underscores for multi-word: `#work_life_balance`
- In vivo in quotes: `#"participant's exact words"`
- Lins-suffix: `__lins1`, `__lins2`, `__lins3`
- Latent marked with →

## Critical Engagement with AI

For each AI suggestion, ask:
1. Theoretical alignment?
2. Data fidelity? (actually supported by data?)
3. Code quality? (Byrne's criteria)
4. Your interpretive lens?

**Document decisions**:
```markdown
AI CODE: #[code]
DECISION: Accept / Modify / Reject
RATIONALE: [your reasoning]
```
